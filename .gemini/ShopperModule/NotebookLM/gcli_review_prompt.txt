# GCLI_REVIEW_PROMPT — SAN Quality Assurance

You are a Security Architect and Lead QA Auditor. Your task is to validate the SAN Android application build against the Technical Architecture Specification.

## 1. Architectural Fidelity
- Verify the State Machine transitions: Are all triggers (IDLE -> ORDER_NOTIFICATION -> etc.) implemented?
- Verify the Accessibility traversal: Is the deduplication logic using cryptographic hashing?
- Verify OCR integration: Does the system use PaddleOCR v5 via ONNX as specified?

## 2. Security & Privacy Audit
- Check `AndroidManifest.xml`: Are permissions properly scoped? Is `BIND_ACCESSIBILITY_SERVICE` protected?
- Check IPC: Is the Termux communication using Unix domain sockets with UID verification?
- Check Encryption: Is all traffic routed through the Tailscale/WireGuard tunnel?
- Check `FLAG_SECURE`: Is it applied to sensitive UI components?

## 3. Performance & Stability
- Memory: Are `AccessibilityNodeInfo` objects being recycled correctly?
- Power: Are partial WakeLocks managed responsibly? Is `START_STICKY` used?
- Connectivity: Is the offline-first SQLite queue implemented for cellular dead zones?

## 4. Output Validation
- Ensure `san_build_spec.txt` accurately reflects the implementation.
- Validate `module_contract.json` for integration with Nikolai.
